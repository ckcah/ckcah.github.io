腾讯云数据库服务器备份下载

这次的腾讯云数据库服务器，是做了每天自动增量备份的，于是直接下载.xb文件。（下图是网上找的，只是为了演示）

重构数据库服务器

数据格式

.xb格式是数据库进行备份产生的，类似于我们平时用的.rar、.tar压缩包。

在一边下载的过程中，一边确定好数据库是mysql 5.7

安装centos8虚拟机

这一步较为简单，默认安装，然后硬盘选择1T的，（后来加了个2T的，解压时觉得不够，又加了个3T，结果发现2T就够了）

安装mysql及解压工具

宝塔及mysql安装

yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh

安装完成后：

在控制面版安装mysql5.7

Percona XtraBackup安装

1.安装Percona yum存储库：

yum install -y https://repo.percona.com/yum/percona-release-latest.noarch.rpm

最后会显示

2.启动存储库：

因为要将Percona XtraBAckup与已安装的Mysql结合使用，所以只需要启用工具存储库：

percona-release enable-only tools

如果只是单纯启动存储库，就可以使用percona-release enable-only tools release

3.安装Xtrabackup 2.4版本

  • MySQL 5.7：请安装Xtrabackup 2.4。
  • MySQL 8.0：请安装Xtrabackup 8.0。

注意：在安装之前，需要先安装依赖。

wget ftp://ftp.pbone.net/mirror/apt.sw.be/redhat/el6/en/x86_64/rpmforge/RPMS/libev-4.15-1.el6.rf.x86_64.rpm

rpm -ivh libev-4.15-1.el6.rf.x86_64.rpm

yum install cmake gcc gcc-c++ libaio libaio-devel automake autoconf bzr bison libtool ncurses5-devel

yum -y install perl perl-devel libaio libaio-devel perl-Time-HiRes perl-DBD-MySQL

最后安装Xtrabackup：

yum install -y percona-xtrabackup-24

安装qpress

yum install qpress -y

解压.xb

cat vv_MySQL_master_backup_20220815000248.xb | xbstream -x -v -C /root/data/

报错：
xbstream: Error writing file './shenqics/LG_GameLogin.ibd.qp' (Errcode: 28 - No space left on device) xbstream: my_write() failed.

这里就是我所说的磁盘空间不足，于是我新增了一块2T的硬盘

1）查看新增磁盘

挂载前：

挂载过程：

空白处为回车

再次进行查看：

可将挂载命令写入开机自挂载文件中：

echo "/dev/nvme0n2 /mnt1 ext4 defaults 0 0" >> /etc/fstab

再次执行cat XXX.xb | xbstream -x -v -C /mnt1/

xtrabackup --decompress --target-dir=/mnt1/data
或
innobackupex --decompress --remove-original ./

–decompress：该选项表示解压 –compress选项压缩的文件。

–remove-original删除原有的.qp文件

添加数据库

解压后的文件与mysql文件结构相同

2Xf2E6aEU7n685eHEbXGYHrmWn2y7a62UWBrtZzodVdD

这是最后一步，但最后没做成功，它显示 数据表崩溃

然后我对数据表进行了修复，显示修复失败

百度以后，说是数据库结构不一样，这三个文件是关键
